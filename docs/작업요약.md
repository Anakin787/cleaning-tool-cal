## 전체 작업 목록

1. 프로젝트 실행 환경 구성 – 앱을 돌리기 위한 기본 세팅
2. Firebase 연결 – 일정·투표 데이터를 클라우드에 저장
3. 일정 참석/불참/미정 – 참석(초록)·불참(빨강)·미정(주황) 버튼으로 응답 선택
4. 날짜·시간 표시 변경 – 날짜는 “2025년 2월 24일”, 시간은 14:30 같은 24시간 형식
5. 투표 확장 – 여러 항목 선택, 익명 투표, 진행 중 새 항목 추가 가능
6. 일정 수정 – 연필 버튼으로 일정 내용 수정
7. 캘린더 – 월별 달력, PC에서는 오른쪽 사이드바에 고정
8. UI 개선 – 버튼 호버 효과, 연결 상태 표시
9. GitHub 배포 – push 시 웹사이트 자동 배포
10. 빌드 오류 수정 – 컴파일 에러 해결
11. 모바일 반응형 – 터치 영역 확대, 화면 크기에 맞게 레이아웃 조정
12. 일정 탭 개편 – 위에 캘린더, 아래 전체 목록, 클릭 시 상세+댓글
13. 일정 댓글 – 일정마다 의견/확답 예정일 등 자유롭게 작성
14. 참/불 이름 – 처음 한 번 이름 선택 후 참석·불참에 이름 표시
15. 홈 개선 – 일정에 수정·삭제 버튼, 투표 개수 표시
16. 투표 보강 – 단일 선택 변경, 마감일, 누가 투표했는지 표시
17. Git 연결 – GitHub 원격 저장소 연결
18. 삭제 확인 모달 – 삭제 전 "삭제하시겠습니까?" 확인창 표시
